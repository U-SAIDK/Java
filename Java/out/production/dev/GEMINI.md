# Java Learning & Reference Project

This project is a comprehensive educational repository for learning and referencing Java programming concepts. It covers a wide range of topics, from basic syntax to advanced object-oriented programming (OOP) principles and system-level features.

## Project Overview

The repository is organized topically, with each directory focusing on a specific Java concept. These files are designed as learning aids, featuring detailed explanations, examples, and memory diagrams within the comments.

### Key Topic Areas

- **Basics**: Data types, variables, operators, type casting, and basic I/O.
- **Control Statements**: Loops (`for`, `while`), `if-else` logic, `switch` cases, and flow control (`break`, `continue`).
- **OOP (Object-Oriented Programming)**: Classes, objects, inheritance, polymorphism, abstraction, and encapsulation.
- **Collections**: `ArrayList`, `HashMap`, `HashSet`, and other data structures.
- **Methods**: Lambda expressions, recursion, overloading, and overriding.
- **Advanced Concepts**: Multithreading, exception handling, generics, and file I/O.
- **Practice**: A dedicated `Practice/src` directory for hands-on implementation of concepts like aggregation and abstraction across multiple files.

## Building and Running

### Prerequisites

- **Java Development Kit (JDK)**: Recommended version 11 or higher.
- **IntelliJ IDEA**: The project includes `.idea` and `.iml` configuration files for seamless use with IntelliJ.

### Running Examples

1.  **Via IDE (Recommended)**:
    - Open the project root in IntelliJ IDEA.
    - Navigate to the desired topic folder.
    - Right-click the `.java` file containing a `main` method and select **Run**.

2.  **Via Command Line**:
    - For Java 11 and above, you can run single files directly:
      ```bash
      java Basics/DataTypes/DataTypes.java
      ```
    - For older versions or multi-file programs, compile and then run:
      ```bash
      javac OOP/Inheritance/Inheritance.java
      java OOP.Inheritance.Inheritance
      ```

## Development Conventions

- **Topical Grouping**: Always place new examples in their respective concept folders or create a new folder if the topic is new.
- **Educational Annotations**: Code should be heavily commented. Use comments to explain *why* something is done, not just *what* is done. Memory diagrams (Stack vs. Heap) are encouraged for reference types.
- **Example Numbering**: Use `// Ex: 1`, `// Ex: 2`, etc., to distinguish between different variations of a concept within the same file.
- **Practice Submodule**: Use the `Practice/src` directory for more complex experiments that require multiple classes interacting across separate files.
- **Naming**: Follow standard Java naming conventions (PascalCase for classes, camelCase for methods and variables).

## Directory Structure Highlights

- `Basics/`: Core language syntax and primitive types.
- `OOP/`: Core principles of Object-Oriented Programming.
- `Practice/src/`: Active development area for more involved coding exercises.
- `MiniProjects/`: Placeholder for small console-based applications.
- `out/`: Compiled `.class` files generated by the IDE.
